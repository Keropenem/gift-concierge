# **1\. コミュニケーション設計**

## **1.1 施策の全体像**

**施策名:** 「Con-TecT 公開ギフト相談」

X上でギフト相談を受け付け、Con-TecTのコア体験である「セレンディピティ×ストーリー×具体的提案」を公開コンテンツとして可視化するテストマーケティング施策。

## **1.2 設計思想: Public-Privateハイブリッドモデル**

**核心原則:** 「種明かしは公開、仕込みは非公開」

Con-TecTの最大の武器である「想定外の共通点の発見 → ストーリー → 意外なギフト提案」という一連の体験を第三者が傲観できる状態にすることで、「自分もやってほしい」という欲求を自然に喚起する。一方で、プロフィール情報や関係性というプライベートな情報を扱うため、ヒアリングは必ずDMで行う。

| フェーズ | チャネル | 内容 | 目的 |
| :---- | :---- | :---- | :---- |
| **Step 1: 募集** | 公開（TL） | 定時募集ポスト | 認知・参加者獲得 |
| **Step 2: 参加** | 公開（TL） | ユーザーがリプライ | エンゲージメント向上 |
| **Step 3: DM誘導** | 公開→DM | 感謝 \+ DMへの誘導リプライ | プライバシー保護 |
| **Step 4: ヒアリング** | 非公開（DM） | 自分・相手・関係性を聴取 | データ収集 |
| **Step 5: 分析・提案生成** | 裏側（運営） | Con-TecTでStep 4〜7実行 | 価値生成 |
| **Step 6: 公開許可** | 非公開（DM） | 匹名化確認 \+ 掲載許可 | コンプライアンス |
| **Step 7: 提案スレッド** | 公開（TL） | 5ツイートの提案スレッド | バイラル拡散 |
| **Step 8: CTA・転換** | 公開（TL） | 次の募集 \+ サービスURL | 循環・獲得 |

## **1.3 ターゲットオーディエンス**

**プライマリ:** 「誰かにプレゼントを贈りたいが、何を選べばいいか悩んでいる人」

**セカンダリ:** スレッドを見て「自分もやってほしい」と感じるギフト感度の高い層

**無料アカウントユーザーへの配慮:** 参加ハードルを「相談したい！」の一言リプライに極限まで下げる。DMが開放されていない場合のフォールバック導線も用意。

## **1.4 投稿タイミング設計**

| 投稿種別 | 時間帯 | 頻度 | 理由 |
| :---- | :---- | :---- | :---- |
| 募集ポスト | 20:00〜21:00 | 毎日（平日） | 仕事後のリラックスタイム。ギフトを考える余裕がある |
| 提案スレッド | 12:00〜13:00 | 遑31〜3件 | 昇休みの高TL閲覧率。読み物として消費される |
| 週末まとめ | 土曜 10:00 | 週次1回 | 週のベスト提案を振り返り。保存・拡散率が高い |

## **1.5 匹名化ルール**

公開コンテンツにおける個人情報保護のため、以下の匹名化ルールを必ず遵守する:

* 具体的な年齢は「30代」「20代後半」のように丸める

* 職業は「クリエイティブ職」「技術職」のようにカテゴリ化

* 名前・会社名・学校名は一切出さない

* 関係性は「学生時代からの親友」「同僚」のように抽象化

* 特定可能なエピソードは、本質を保ちつつ改変する

## **1.6 アカウント運用方針**

**アカウント種別:** Premium認証済み個人アカウント

個人アカウントであることを強みに変える。「運営者自身がギフト選びに悩んだ経験から作ったサービス」というストーリーをプロフィールで語ることで、企業アカウントよりも「人間味」と「信頼感」を両立できる。Premium認証により、DM開放が容易であり、リプライの表示優先度も高い。

# **3\. KPI・測定計画**

## **3.1 ファネル全体のKPI設計**

施策の効果を測定するため、ファネルの各段階にKPIを設定する。

| ファネル段階 | KPI | 計算式 | 目標値（1ヶ月後） | 測定方法 |
| :---- | :---- | :---- | :---- | :---- |
| 認知 | 募集ポストImp | インプレッション数 | 5,000 / 投稿 | X Analytics |
| 興味 | エンゲージメント率 | (いいね+RT+リプ) / Imp | 3%以上 | X Analytics |
| 参加 | 相談リプライ数 | 「相談したい」リプライ数 / 日 | 5〜10件 / 日 | 手動カウント |
| DM転換 | DM開始率 | DM開始数 / 相談リプライ数 | 70%以上 | 手動カウント |
| 完了 | ヒアリング完了率 | 3通完了数 / DM開始数 | 60%以上 | 手動カウント |
| 公開 | 掲載許可率 | 掲載OK数 / 提案完了数 | 80%以上 | 手動カウント |
| 拡散 | スレッドRT数 | 1スレッドあたりのRT総数 | 50 RT / スレッド | X Analytics |
| 転換 | サイトクリック率 | URLクリック / スレッドImp | 2%以上 | UTM \+ GA4 |
| 獲得 | 会員登録数 | 新規会員 / 週 | 10件 / 週 | Supabase |

## **3.2 スレッドコンテンツの品質指標**

| 指標 | 定義 | 目標 |
| :---- | :---- | :---- |
| 保存率 | ブックマーク数 / Imp | 1%以上（通常の3倍が基準） |
| 引用RT率 | 引用RT / 全RT | 30%以上（読後の感想付きRT） |
| スレッド完読率 | 最終TweetのImp / 初回TweetのImp | 40%以上 |
| 商品URLクリック率 | 商品URLクリック / スレッドImp | 1.5%以上 |

## **3.3 UTMパラメータ設計**

全てのリンクに以下のUTMを付与し、流入元を正確にトラッキングする。

| パラメータ | 値 | 例 |
| :---- | :---- | :---- |
| utm\_source | x (twitter) | x |
| utm\_medium | social | social |
| utm\_campaign | gift\_consultation | gift\_consultation |
| utm\_content | スレッド番号\_ツイート番号 | thread012\_tweet3 |
| utm\_term | 商品カテゴリ | glasspen |

## **3.4 週次レポート項目**

毎週月曜日に以下を集計し、トレンドを把握する:

* 募集ポストの平均エンゲージメント率の推移

* 相談リプライ数 → DM開始 → 完了 → 掲載OKのファネル数値

* スレッド別のRT数・保存数・クリック数のランキング

* サイト流入数と会員登録転換率

* 最もRTされたスレッドの共通点分析（どんなストーリーが響いたか）

## **4.2 相談対応チェックリスト**

各1件の相談について、以下のステータスを管理する:

| ステータス | 定義 | 次アクション |
| :---- | :---- | :---- |
| **① リプライ受付** | 相談リプライを確認 | DM誘導リプライを送信 |
| **② DM誘導済** | DM誘導リプライ済み | DM 1通目を送信 |
| **③ ヒアリング中** | DMでやりとり中 | 3通完了までフォロー |
| **④ Con-TecT実行中** | ヒアリング完了、分析中 | 提案結果をDMで共有 |

| **⑥ スレッド投稿済** | 公開済み | エンゲージメント対応 |
| **⑦ 完了** | 全プロセス完了 | データ記録・振り返り |

## **4.4 トラブルシューティング**

| トラブル | 対応方法 |
| :---- | :---- |
| DM未開放ユーザー | フォローバックテンプレートでDM開放 or 相互フォローを案内 |
| DMで返信がない | 24時間後にリマインダー。1回のみ。それでも返信なしならクローズ |
| 個人情報を公開リプで書かれた | 即座に「DMでお願いします」と誘導。公開リプは削除依頼しない（ユーザーの判断に任せる） |
| 掲載NGだった | 「承知しました。提案内容はそのままお使いください」と感謝を返信 |
| 提案に不満 | 「別の角度でもう10件提案しましょうか？」と再提案を提示 |
| 商品URLがリンク切れ | 代替商品を提案 or 「商品名で検索してください」と案内 |
| スレッドがバズらない | 投稿曜日・時間帯を変更。他スレッドとの差分を分析 |

## **4.5 エスカレーションポリシー**

以下の場合は対応を中止し、記録に残す:

* 誣謗中傷・脱法行為の示唆がある場合

* 明らかに商業目的（アフィリエイト詐欺等）の相談の場合

* 個人情報を意図的に拡散しようとする行為の場合

## **4.6 週次振り返りテンプレート**

毎週月曜日に以下を振り返る:

1. 先週のファネル数値（リプライ→DM→完了→公開→拡散）

2. 最も反応が良かったスレッドとその要因分析

3. 最も反応が悪かったスレッドとその改善仮説

4. 募集ポストのパターン別成績比較

5. DMヒアリングでの離脱ポイント分析

6. 来週の改善アクション（3つまで）
