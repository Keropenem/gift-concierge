# **2\. プロダクトビジョン**

## **2.1 サービス名の由来**

**Con-TecT** — 贈り相手と自分の**Context（人生の文脈）**をマッチして購入できるEC。ギリシャ語の**Tect（大工・技術人）**の意味も含み、モノづくり企業やクリエイターを応援する思想を表現。

## **2.2 ミッションステートメント**

**「贈り物は、二人の物語から生まれる」**

贈り手と受け取り手の人生の文脈・関係性から「見えない共通点」を見出し、そのストーリーに合った唄一無二のプレゼントを提案するECサービス。製造元のECを優先的に紹介し、モノづくりの作り手と贈り手を直接つなぐ。

## **2.3 コア体験（バリュープロポジション）**

| コア体験 | 説明 |
| :---- | :---- |
| セレンディピティ | 贈り手が想像できていない相手との共通点を見出し、想定外のプレゼントを提案 |
| 深い「想い」の実現 | プレゼントの本質である「相手のことを考える」を深いレベルで実現 |
| 贈り方の演出 | プレゼントの渡し方・添えるメッセージまで提案 |

# **3\. ターゲット・ビジネスモデル**

## **3.1 ターゲットユーザー**

初期MVPでは特定セグメントに限定せず、「誰かにプレゼントを贈りたい全ての人」を対象とする。利用データの蓄積を通じて、コアユーザー層を特定する。

## **3.2 収益モデルロードマップ**

| フェーズ | 収益モデル | 概要 |
| :---- | :---- | :---- |
| **Phase 1（MVP）** | 無料 | ユーザー獲得・プロダクト検証に集中。製造元ECへの送客（アフィリエイト型） |
| **Phase 2（グロース）** | 販売・決済手数料 | 提携先製造元の商品をサービス上で購入可能に。手数料率20%前後 |
| **Phase 3（拡大）** | 手数料 \+ 独自DB | 自社提携先の独自商品DBを構築。キュレーション型のECへ進化 |

## **3.3 商品価格帯**

MVP段階では価格帯の制限を設けない。LLMがユーザーの状況・文脈に応じて最適な価格帯の商品を自動判断し提案する。

# **4\. 機能要件**

## **4.1 ユーザーフロー全体像**

1. トップページにアクセス（Google検索風のミニマルUI）

2. Chat UIの検索窓に自由入力 or クリックで会話開始

3. LLMがStep 1〜Step 3を対話形式でヒアリング

4. Step 4〜Step 7を自動実行し、結果を表示

5. 提案商品の下に、分析から見える嗜好性に関連する商品5つを追加提案

6. 未会員の場合、提案結果の下に会員登録訴求を表示

## **4.2 画面一覧・機能詳細**

### **4.2.1 トップページ**

| 要素 | 仕様 |
| :---- | :---- |
| レイアウト | Google検索ホーム風のミニマルUI。中央にロゴ、その下にChat入力欄 |
| 検索窓 | プレースホルダー: 「誰に、どんな贈り物をしたいですか？」 |
| ナビゲーション | 右上に会員登録 / ログインボタン |
| フッター | 利用規約・プライバシーポリシーリンク |

### **4.2.2 チャット画面（メイン機能）**

| 要素 | 仕様 |
| :---- | :---- |
| チャットUI | メッセージバブル形式。ユーザー入力は右側、LLM応答は左側 |
| LLMヒアリング | Step 1〜3を対話形式で実行（後述のLLM要件参照） |
| 提案結果表示 | Step 4〜7の結果をカード形式で表示。商品画像・価格・製造元URL付き |
| 関連商品提案 | 提案結果の下に、嗜好性に基づく商品5つをカルーセル表示 |
| 会員登録訴求 | 未会員のみ: 提案結果の下に「次回から入力不要」の説明 \+ 会員登録ボタン |

### **4.2.3 会員登録画面**

**登録項目:**

| フィールド | 型 | 必須 | 備考 |
| :---- | :---- | :---- | :---- |
| メールアドレス | メール | ○ | ログインIDとして使用 |
| パスワード | パスワード | ○ | 最低8文字、英数字混合 |
| 年齢 | 数値 | ○ | 基本プロフィール |
| 性別 | 選択 | ○ | 男性/女性/その他/回答しない |
| 職業 | テキスト | ○ | 自由入力 |
| 関心事 | テキスト | ○ | 複数入力可（タグ形式） |
| 得意なこと | テキスト | ○ | 複数入力可（タグ形式） |

### **4.2.4 会員専用機能**

* 基本プロフィールの自動反映: Step 1のヒアリングをスキップし、Step 2から開始

* 現在の状況ヒアリング: 基本プロフィール以外の「今の気持ち」「最近の出来事」などは都度ヒアリング

* 提案商品のクリック履歴保存: 興味関心のデータ蓄積に活用

* プロフィール編集機能: マイページから基本情報の更新が可能

# **5\. LLM要件（Narrative Gift Synthesis）**

## **5.1 使用モデル**

**Gemini**（Google AI）を使用。Web検索機能（Grounding with Google Search）を活用し、商品検索・製造元URLの取得をリアルタイムで実行。

## **5.2 プロンプト設計: 7ステッププロセス**

**システムプロンプト:** 「あなたは世界最高のギフトコンシェルジュであり、優れたストーリーテラーです」

| ステップ | フェーズ | 内容 | 実行方法 |
| :---- | :---- | :---- | :---- |
| Step 1 | Profile | 自分自身の基本プロフィール入力 | ユーザー入力（会員はスキップ） |
| Step 2 | Target | 相手の基本プロフィール入力 | ユーザー入力 |
| Step 3 | Context | 二人の関係性を象徴するキーワード5つを定義 | ユーザー入力 |
| Step 4 | Analysis | 見えない共通点と背景の感情を言語化 | LLM自動実行 |
| Step 5 | Narrative | 共通点を軸にした心に響く短い物語を執筆 | LLM自動実行 |
| Step 6 | Proposal | 物語を象徴する贈り物を提案（商品名 \+ 製造元URL） | LLM \+ Gemini Web検索 |
| Step 7 | Action | 渡す際の演出と添えるメッセージを提案 | LLM自動実行 |

**重要ルール:** Step 1〜Step 3は必ずユーザーへの問いかけとして実行。Step 4〜Step 7はLLMが自動実行する。

## **5.3 商品検索ロジック**

* GeminiのGrounding with Google Searchを使用し、リアルタイムで商品を検索

* 製造元・クリエイターの公式ECサイトのURLを優先的に取得

* 商品情報（名前・価格・画像・説明・URL）を構造化してフロントに返却

* 商品が見つからない場合のフォールバック（カテゴリ提案）を実装

## **5.4 関連商品レコメンドロジック**

Step 4のAnalysis結果から抽出された嗜好性・共通点キーワードを基に、メイン提案とは異なる角度から5つの関連商品を提案する。これにより、ユーザーに幅広い選択肢を提供する。

# **11\. リスク・課題・対策**

| リスク | 影響度 | 対策 |
| :---- | :---- | :---- |
| Gemini Web検索の商品URL精度 | **高** | リンク切れ検知・フォールバック提案を実装。将来的に自社DBで補完 |
| LLM応答の品質バラツキ | **高** | プロンプトの反復チューニング、品質モニタリング体制 |
| セッション完了率の低下 | **中** | ヒアリングのステップ数最適化、UX改善 |
| 個人情報の取り扱い | **中** | プライバシーポリシー策定、データ最小化原則 |
| APIコストの増大 | **低** | キャッシュ戦略、レートリミット設定 |

# **12\. MVPスコープ境界**

## **12.1 MVPに含むもの（In Scope）**

* Chat UIによる7ステップのギフト提案フロー

* Gemini Web検索による商品検索・製造元URL取得

* メール/パスワード認証・会員登録

* 基本プロフィールの保存・自動反映

* 提案商品のクリック履歴記録

* 嗜好性に基づく関連商品5つの追加提案

* レスポンシブ対応（モバイルブラウザ対応）

## **12.2 MVPに含まないもの（Out of Scope）**

* サービス上での決済・購入機能（Phase 2）

* 自社商品DBの構築（Phase 3）

* OAuthソーシャルログイン（Google, LINE等）

* ネイティブモバイルアプリ

* 多言語対応

* レコメンドエンジン（協調フィルタリング等）

* プッシュ通知・リマインダー機能

# **13\. 付録: ユーザーストーリー（使用例）**

**ペルソナ:** 30歳の男性会社員が、学生時代からの親友（32歳女性、デザイナー）に誕生日プレゼントを贈りたい。

1. トップページで「友人に誕生日プレゼントを贈りたい」と入力

2. LLMが自分のプロフィールをヒアリング（Step 1）

3. 相手のプロフィールをヒアリング（Step 2）

4. 関係性のキーワードを定義（Step 3）: 「大学の美術部」「一緒に見た展覧会」「旅行」「写真」「カフェ」

5. LLMが「二人が共有する美意識と創造性」という見えない共通点を発見（Step 4）

6. 二人の物語を執筆（Step 5）

7. 「伝統工芸のガラスペン」を提案（Step 6） — 製造元工房の公式EC URL付き

8. 渡す際の演出とメッセージを提案（Step 7）

9. 「あなたの嗜好性に合う商品」とし5つの関連商品を提案

10. 未会員の場合: 会員登録訴求を表示
